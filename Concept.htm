Concept

In TriviaTime, we are changing the concept of the current gameplay to offer an improved experience.

Question

<Glime> .001. Alcohol: Name That drink: Kosak’s ____?

<Glime> 1st Hint: ****

The goal of this new theme is to bold the most important information to make the question output as simple as possible. A change from the current theme is that we will not include the question score. Hints separated by ten seconds, instead of 15.

KAOS

<Glime> .K001. What are some popular ice cream flavors? 14 possible answers possible

<Glime> 1st Hint: [***** ****] [**** ******** ****] [** ****** *****] [***** ****] [**** ******** ****] [** ****** *****] [***** ****] [**** ******** ****] [** ****** *****] [***** ****] [**** ******** ****] [** ****** *****] [**** ******** ****] [** ****** *****]

Again, here, only the important information is included and in bold. Point values are left out. After a question is answered and before the next hint, a command will be available to view the remaining answers (currently: ,).

Points

To even the playing field, we are changing the way points work. Each question will be worth the same amount of points, instead of a point range (such as 100 – 900). A bonus for questions will be a set amount. KAOS answers will be the same way. The KAOS bonus will also be a set amount. This offers a far advantage to all players. Bonus questions are not planned.

Database(s)

We plan on keeping track of players’ scores by their usernames. Should they want to change usernames and keep their scores, they will need to register to the bot. Supybots are already capable of keeping track of users across nicknames and hosts (per owner’s configuration), so why reinvent the wheel? SQLite will be used for the database(s).

Keeping track of questions

Store question in an SQL file. Create a system of assigning number to the question. That question number will only be used for reporting and when the question is displayed by the bot. If the question is reported, it will be assigned a new number for reporting. When edited, it will use the new report number.

Streaks

Once a player begins a streak, the point value will increase by .01% on each question for that player only. For example, if 500 points are awarded for each question, a streak of two would reward 505 points, then 510, 515, 520, 525, 530, etc. Up to 2x its original value. If the streak is broken by another player, that player is awarded 500 points and can start a new streak for themselves. If the game stops, the streak will still continue when it starts again. Alternative approach: increase by just one point only each time. Other ideas: After a streak of five, a bonus will be issued as well as points increasing each time.

On the Web (Reporting, Fixing, and Web Stats)

Reports will automatically be published to a PHP webpage. Once the report has been fixed, it will be automatically moved to a Fixed page. The report will include who reported it, and the fixed version will include who fixed it. Counts will be kept public and live of how many questions each player has reported and fixed. Web Stats will also be kept on a PHP webpage by pulling the information directly from the database.

Reporting

If a report is done properly using regex, it will be corrected and saved automatically. As the question cycle comes to an end users will be reminded to fix their reports. Reports will be removed once they have been fixed and verified by someone else.

Enhanced Administration

Unlike BogusTrivia, TriviaTime will make it easier to make changes to the bot. Settings can be changed within the supybot online and have changes applied in real time. They will also be able to increase/decrease player scores, migrate player scores, delete players, change player level, award badges and prohibit a player from earning points.

Minor details

The game will no longer start automatically. It will stop after ten minutes of silence in the channel. There will no longer be an on-join greeting. Flood protection will be enabled by default.

New KAOS

This is still an idea. Instead of offering all possible answers at the beginning of the KAOS, instead, only ten (randomly chosen) answers will be available. As the players near getting all ten (such as 8 answered), more answers will be offered and the time will be expanded.

Levels

This is still an idea. After a certain number of points have been reached, or after a certain number of questions have been answered, the player will move up a level. Also consider: increase streak multiplier as you level up? How many points should a player be awarded?

Badges

This is still an idea. Doing certain actions will award the player a badge. Badges are permanent and can be viewed on the website. Ideal uses are for fixing/adding/editing questions, being active, being around for a while, reaching a certain amount of points.
